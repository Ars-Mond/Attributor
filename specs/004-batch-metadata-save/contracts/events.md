# Contract: Unified Event Contract (broadcast events)

The single source of truth for backend→frontend **broadcast** events: names + payload types in
one Rust module; emission stays in the owning domain (FR-012/FR-013). Per-operation progress is
NOT here — it uses the Channel (see `channel-protocol.md`), per FR-014.

## Source of truth: `src-tauri/src/events.rs`

```rust
pub const FOLDER_CHANGED: &str = "folder-changed";
pub const THUMBNAIL_READY: &str = "thumbnail-ready";

#[derive(Serialize, Clone)]
#[serde(rename_all = "camelCase")]
#[cfg_attr(test, derive(ts_rs::TS))]
#[cfg_attr(test, ts(export, export_to = "../../src/lib/generated/events.d.ts"))]
pub struct FolderChanged { pub path: String }

#[derive(Serialize, Clone)]
#[serde(rename_all = "camelCase")]
#[cfg_attr(test, derive(ts_rs::TS))]
#[cfg_attr(test, ts(export, export_to = "../../src/lib/generated/events.d.ts"))]
pub struct ThumbnailReady { pub path: String }
```

`BatchProgress` / `ItemStatus` (channel types) carry the same `ts-rs` export attributes so they
land in the same generated file.

## Emission (stays in domain modules)

| Event | Emitter | Call |
|-------|---------|------|
| `THUMBNAIL_READY` | `folder/pipeline.rs` | `app.emit(events::THUMBNAIL_READY, events::ThumbnailReady{path})` |
| `FOLDER_CHANGED` | `folder/watch.rs` | `app.emit(events::FOLDER_CHANGED, events::FolderChanged{path})` |

`folder-changed` payload is **promoted** from a bare `String` to `FolderChanged{path}` so no
untyped crossing remains. Its only current listener (`FilesPanel.svelte`) ignores the payload →
safe migration with no behavior change (FR-015).

## Frontend consumption: `src/lib/events.ts`

```ts
import {listen} from "@tauri-apps/api/event";
import type {FolderChanged, ThumbnailReady} from "./generated/events";

export const EVENT = {folderChanged: "folder-changed", thumbnailReady: "thumbnail-ready"} as const;

export function listenEvent<T>(name: string, cb: (payload: T) => void) {
    return listen<T>(name, (e) => cb(e.payload));
}
```

`FilesPanel.svelte` switches its `listen("thumbnail-ready"|"folder-changed")` calls to
`listenEvent<ThumbnailReady|FolderChanged>(EVENT.…, …)` using the generated types.

## Drift guard (FR-016)

- `src/lib/generated/events.d.ts` is generated by `ts-rs` and **committed**.
- `src-tauri/tests/events_contract_test.rs` re-exports to a temp dir and asserts byte-equality
  with the committed file (line endings normalized for §IV). A Rust name/payload change without
  regeneration fails this test; a stale frontend type fails `svelte-check`.
- Regeneration command documented in `quickstart.md`.
